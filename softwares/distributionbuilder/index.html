DistributionBuilder

Subjective Probability Elicitation Made Easy

1. Introduction

A. Motivation

In 'Lay understanding of probability distributions' (2014) Daniel Goldstein and David Rothschild have highlighted the benefits of using graphical interfaces called distribution builders to study subjective probabilities, perceptions of frequency, and confidence judgements.

However, the implementation of such distribution builders in online studies is not straightforward, as survey platforms do not implement them. The DistributionBuilder library was created to address this issue, and make the implementation of distribution builders easy and accessible to researchers. The library is user-friendly, and requires very little programming knowledge to be used.

2. Installation

You have several ways of adding DistributionBuilder to your experiments:

A. Qualtrics

If you prefer starting from a minimal working example and tweaking it, here is a fully functional integration in the form of a .qsf file that you can import in Qualtrics.

Otherwise, just follow these simple steps:

  1. Navigate to the "Look and Feel" section of your survey, and click on the "Advanced" tab
  2. Edit the "Header" section, and add the following lines to load the library script and the library styles: 
<link rel="stylesheet" href="https://cdn.rawgit.com/QuentinAndre/DistributionBuilder/master/distributionbuilder.css">
<script src="https://cdn.rawgit.com/QuentinAndre/DistributionBuilder/master/distributionbuilder.min.js"></script>

That's it! You can now add your own distribution builders in the "Custom Javascript" section of your Qualtrics questions. For examples on how to use it, refer to the instructions on this page.

B. Stand-Alone Website

If you are hosting your experiment on a separate website, here is how to add the script:

  1. Download the Library using the button at the top of the page.
  2. Add the Javascript file distributionbuilder.min.js and the CSS file distributionbuilder.css to your html code:
<link rel="stylesheet" href="distributionbuilder.css">
<script src="distributionbuilder.min.js"></script>
  1. That's it! You can now add your own distribution builders to your webpage. No need to include jQuery! DistributionBuilder automatically injects it, and makes it accessible under the $j alias. Instead of typing $('#Hello'), type $j('#Hello').

4. Customization

The DistributionBuilder object is created in different steps:

  • Initialization (defining the object and its properties)
  • Render (tweaking where and how it should appear on the page)
  • Label (choosing what the labels for the buckets should look like)

Each steps offer opportunities to customize the DistributionBuilder.

A. Initialization

The first step in creating a Distribution Builder is to initialize the object:

myDistBuilder = new DistributionBuilder({})

This function takes the following parameters:

  • nBuckets, (default: 10): The number of buckets (columns) to which balls can be allocated.
  • nRows, (default: 10): The number of rows of the distribution builder (i.e. the maximum number of balls that can be allocated to a certain bucket).
  • minVal, (default: 0): The value corresponding to the first bucket (the smallest value).
  • maxVal, (default: 10): The value corresponding to the last bucket (the largest value).
  • nBalls, (default: 10): The total number of balls to allocate.
  • onTouch, (default: function () {}): A JavaScript function that will be called every time the user clicks a button of the distribution builder to add/remove a ball.
  • onChange, (default: function () {}): A JavaScript function that will be called every time the user successfully changes the allocation of balls (i.e. when the add/remove action is performed on non-filled/non-empty bucket, and when there are still balls available to allocate).
Demo:
  • Output
  • Javascript
  • HTML
var distbuilder = new DistributionBuilder({
    nRows: 20,
    nBuckets: 20,
    minVal: 0,
    maxVal: 100,
    nBalls: 20,
    onTouch: function () {
        console.log('Distbuilder was touched!')
    },
    onChange: function () {
        console.log('Distbuilder was updated!')
    }
    });
distbuilder.render('targetdiv');
distbuilder.labelize();

B. Render

After the DistributionBuilder object is initialized, you must call another method to indicate where you want to see it displayed on the page, and how it should be displayed.

DistributionBuilder.render()

This method requires the argument target, and you can optionally supply the order argument to tweak the appearance of the Distribution Builder.

  • target (required argument): The html id attribute of the element in which the distribution builder should be displayed. For best results, this element should be a <div> with a fixed width.
  • order, (default: ‘grid-labels-buttons’): The order in which the elements of the Distribution Builder should be rendered. If you want for instance the labels to appear above the grid, you should specify ‘labels-grid-buttons’.

We can for instance use this function to place the labels at the top of the DistributionBuilder:

Demo:
  • Output
  • Javascript
  • HTML
var distbuilder = new DistributionBuilder({
    nRows: 10,
    nBuckets: 20,
    minVal: 0,
    maxVal: 100,
    nBalls: 10,
    onTouch: function () {
        console.log('Distbuilder was touched!')
    },
    onChange: function () {
        console.log('Distbuilder was updated!')
    }
    });
distbuilder.render('targetdiv', 'labels-grid-buttons');
distbuilder.labelize();

C. Label

The last step adds some labels to the buckets of your DistributionBuilder:

DistributionBuilder.labelize()

By default, the Distribution Builder automatically creates evenly spaced labels, using the distance between the minimum and maximum value, and the number of buckets, as follow:

step = (maxValue - minValue)/nBuckets;
baseValue = minValue + step/2;
labels = [baseValue + step*0, baseValue + step*1, ..., baseValue +step*(nBuckets-1)]

Calling the method DistributionBuilder.labelize() without arguments will display those labels.

Alternatively, you can customize the labels using the following arguments:

  • labels, an array of length nBuckets. Supplying this argument will override the default labels.
  • prefix, a string (e.g. ‘$’) which you would like to see prepended to all the labels.
  • suffix, a string (e.g. ‘€’) which you would like to see appended to all the labels.

The prefix and suffix arguments are always applied, regardless of whether you specify custom labels using the label argument.

Demo:
  • Output
  • Javascript
  • HTML
var distbuilder = new DistributionBuilder({
    nRows: 10,
    nBuckets: 20,
    minVal: 0,
    maxVal: 100,
    nBalls: 10,
    onTouch: function () {
        console.log('Distbuilder was touched!')
    },
    onChange: function () {
        console.log('Distbuilder was updated!')
    }
    });
distbuilder.render();
distbuilder.labelize({
    prefix: '~',
    suffix: '€'
});

5. Methods for Accessing Data

After setting up the Distribution Builder, its internal state can be conveniently accessed through three methods:

  • DistributionBuilder.getDistribution() returns the current allocation of balls in the form of an array of length nBuckets. If there are four buckets, and the user has allocated two balls to the first bucket, one to the third one, and zero to the second and fourth, the method will return [2, 0, 1, 0].

  • DistributionBuilder.getRemainingBalls() returns the number of balls that have not been allocated yet.

  • DistributionBuilder.isComplete() returns true is the user has allocated all the balls, and false otherwise.

The following examples demonstrate how those methods can be used in your experiments and studies.

A. Balls Allocated

This can be achieved by creating two <div>s called “BallsLeft” and “BallsAllocated”, and assigning a function to the onChange argument of the Distribution Builder initialization function. This function will read the number of balls allocated and the number of balls left, and update the <div>s everytime those values change.

Demo:
  • Output
  • Javascript
  • HTML
You have 10 balls left.
You have allocated 0 balls.
var n_balls = 10;
$j('#BallsLeft').text('You have ' + n_balls + ' balls left.');
$j('#BallsAllocated').text('You have allocated ' + 0 + ' balls.');
var distbuilder = new DistributionBuilder({
    minVal: 0,
    maxVal: 100,
    nRows: 10,
    nBuckets: 20,
    nBalls: n_balls,
    onChange: function () {
        var remainingballs = this.getRemainingBalls();
        var ballsallocated = n_balls - this.getRemainingBalls();
        $j('#BallsLeft').text('You have ' + remainingballs + ' balls left.');
        $j('#BallsAllocated').text('You have allocated ' + ballsallocated + ' balls.');
    }
});
distbuilder.render('targetdiv');
distbuilder.labelize();

<div id='LabelContainer'>
    <div id='BallsLeft'>You have X balls left.</div>
    <div id='BallsAllocated'>You have allocated X balls.</div>
</div>

B. Validation

You probably want the participants to your studies to allocate all the balls before they are able to validate their distribution. It can be done by creating a button, and only enabling it when all the balls have been allocated:

  • Create a button called “SubmitDistribution” with disabled=true. When clicked, this button will display "Distribution Validated!".
  • Assign a function that will enable or disable the button to the onChange argument of the DistributionBuilder:
    • When DistributionBuilder.isComplete() returns false, the button will be disabled.
    • When DistributionBuilder.isComplete() returns false, the button will be enabled.

This can be used to ensure that the distribution is fully specified before the respondents can proceed to the next page.

Demo:
  • Output
  • Javascript
  • HTML
var distbuilder = new DistributionBuilder({
    minVal: 0,
    maxVal: 100,
    nRows: 10,
    nBuckets: 20,
    nBalls: 10,
    onChange: function () {
        if (this.isComplete()) {
            $j('#SubmitDistribution').attr('disabled', false)
        } else {
            $j('#SubmitDistribution').attr('disabled', true)
        }
    }
});
$j('#SubmitDistribution').click(function() {alert('Distribution Validated!')});
distbuilder.render('targetdiv');
distbuilder.labelize({});

<div style='text-align: center;margin-bottom: 3px;'>
    <button class='btn btn-default' id='SubmitDistribution' disabled='true'>Submit Distribution</button>
</div>
<div id='targetdiv'></div>

D. Saving Data in Qualtrics

You will often want the Distribution Builder to be part of larger survey on Qualtrics, and store the participant's distribution in a variable. Here is one way to do it:

  • In the "Survey Flow" section of your survey, create a new Embedded Data field at the beginning of your survey flow called "MyDistributionResult", or any other name that you want to use. Leave the value empty for now: this is where the results will be stored
  • You can now use the Qualtrics.SurveyEngine.setEmbeddedData() function provided by Qualtrics to store a value in the "MyDistributionResult" variable. The syntax is: Qualtrics.SurveyEngine.setEmbeddedData("MyDistributionResult", "TheStringYouWantToStoreHere")
  • Since this Qualtrics function requires a string, but the method DistributionBuilder.getDistribution() returns an array, you must first convert this array into a string. To do so, simply use DistributionBuilder.getDistribution().join(). This will join the elements of the array by commas, and you will be able to store it in Qualtrics.

Building upon the previous example once again:

Demo:

  • Javascript
  • HTML
var distbuilder = new DistributionBuilder({
    minVal: 0,
    maxVal: 100,
    nRows: 10,
    nBuckets: 20,
    nBalls: 10,
    onChange: function () {
        if (this.isComplete()) {
            $j('#SubmitDistribution').attr('disabled', false)
        } else {
            $j('#SubmitDistribution').attr('disabled', true)
        }
    }
});
distbuilder.render('targetdiv');
distbuilder.labelize({});
$j('#SubmitDistribution').click(function () {
    var results = distbuilder.getDistribution().join();
    Qualtrics.SurveyEngine.setEmbeddedData('MyDistributionResult', results);
    var message = 'The function 'Qualtrics.SurveyEngine.setEmbeddedData('MyDistributionResult', ';
    message +=  results + ') was called. Your data would have been stored in Qualtrics!';

6. CSS Structure

A DistributionBuilder object consists of three elements:

  • The grid, an matrix of cells in which the balls allocated are displayed.
  • The labels, a line matching each bucket to a value.
  • The buttons, which allow the user to increment or decrement a bucket.

If you want to customize the appearance of the distribution builder, the full list of classes defined by those elements can be found in the distributionbuilder.css file.

The following HTML tree will help you understand the how the DistributionBuilder and its elements are rendered on screen:

7. More Information

The first distribution builder was developed by Sharpe, Goldstein and Blythe (2000), and was featured later in Goldstein, Johnson and Sharpe (2008) and Delavande and Rhowedder (2008).

The library DistributionBuilder is published under the MIT license. The code is written in ES2016 Javascript, and is transpiled in legacy code for cross-browsers compatibility. It relies on jQuery, and uses some elements of Bootstrap for styling.
